# Revmamed

Внутренняя система медицинского центра для администраторов и медицинского персонала.

Проект решает практическую проблему операционной работы клиники: вместо разрозненных таблиц, ручного учета и несвязанных процессов в одном приложении собраны пациенты, расписание врачей, запись на прием, лечебные сценарии, документы и служебные операции.

Это продуктовый проект, в котором важны не только модели и формы, но и удобство ежедневной работы сотрудников, устойчивость бизнес-процессов и снижение количества ручных ошибок.

## Контекст

В клинике есть несколько параллельных потоков работы:

- администраторы ведут пациентов и записывают их на прием;
- врачи работают с назначениями и историей посещений;
- персоналу нужны документы, отчеты и быстрый доступ к связанным данным;
- расписание, услуги, цены и записи должны быть согласованы между собой.

Без единой системы такие процессы быстро распадаются на ручную координацию, дублирование данных и ошибки в записи. Этот проект закрывает именно этот слой операционных задач.

## Что делает система

### 1. Централизует данные о пациентах

- хранит карточки пациентов;
- поддерживает редактирование и просмотр ключевых данных;
- помогает работать с резервом и листом ожидания;
- поддерживает внутренние операции импорта и экспорта.

### 2. Упрощает запись и управление расписанием

- позволяет вести расписание врачей и таймслоты;
- поддерживает обычные и процедурные приемы;
- учитывает связанные сценарии записи и изменения приемов;
- снижает риск накладок и ручной путаницы в расписании.

### 3. Поддерживает работу врача после записи

- хранит назначения и связанные лечебные записи;
- дает возможность продолжать или копировать сценарии лечения;
- связывает прием, пациента и медицинский контекст в одном потоке.

### 4. Закрывает внутренний документооборот

- формирует документы по шаблонам;
- поддерживает служебные печатные сценарии;
- помогает персоналу быстрее готовить типовые документы без ручной сборки.

### 5. Учитывает реальные роли пользователей

- разделяет сценарии для администратора, медицинского администратора и врача;
- ограничивает критические действия через права доступа;
- делает интерфейс более безопасным для повседневной работы.

## Моя роль как разработчика

В этом проекте моя работа была не про “собрать CRUD”, а про разработку прикладной системы под реальные процессы:

- проектирование структуры данных под предметную область клиники;
- реализация бизнес-логики вокруг пациентов, приемов, расписания и лечения;
- создание серверных сценариев для сложных форм и связанных операций;
- поддержка внутренних служебных задач: документы, экспорт, импорт, отчеты;
- развитие проекта как рабочего инструмента, который должен быть полезен команде каждый день.

## Структура проекта

- `timetable/` - расписание, врачи, таймслоты, отчеты и справочники;
- `patients/` - пациенты, резерв, waitlist, импорт, экспорт и связанные сценарии;
- `appointments/` - приемы, процедурные приемы и логика записи;
- `treatment/` - назначения и лечебные сценарии;
- `users/` - пользователи, профили, авторизация и права доступа;
- `config/` - настройки проекта.

## Локальный запуск

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

Создайте `.env`:

```env
SECRET_KEY=change-me
DEBUG=True
DATABASE_NAME=medcenter
DATABASE_USER=postgres
DATABASE_PASSWORD=postgres
DATABASE_HOST=localhost
DATABASE_PORT=5432
ENABLE_HTTPS_SECURITY=False
```

Запуск:

```bash
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```
